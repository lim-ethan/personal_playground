# LLM 공부 2편: Word2Vec과 단어 임베딩

## 0. 이번 편의 목표

이번 편에서는 LLM으로 가기 전 단계에서 매우 중요한 개념인 **단어 임베딩(word embedding)** 을 다룬다.

LLM을 제대로 이해하려면 먼저 이 질문에 답할 수 있어야 한다.

> 컴퓨터는 단어의 의미를 어떻게 숫자로 표현할까?

LLM은 텍스트를 직접 이해하지 않는다.  
모든 문장은 먼저 숫자로 바뀌고, 그 숫자들이 모델 내부에서 계산된다.

그래서 Word2Vec은 LLM의 직접적인 조상은 아니지만, **“언어를 벡터 공간에 표현한다”** 는 관점을 널리 퍼뜨린 매우 중요한 기술이다.

---

## 1. Word2Vec 이전에는 단어를 어떻게 표현했나?

초기 NLP에서는 단어를 주로 다음과 같은 방식으로 표현했다.

```text
문장 → 단어 분리 → 단어 ID 부여 → 통계 기반 처리
```

예를 들어 다음과 같은 단어 목록이 있다고 해보자.

```text
[사과, 바나나, 컴퓨터, 보안, 악성코드]
```

각 단어에 ID를 줄 수 있다.

```text
사과 = 1
바나나 = 2
컴퓨터 = 3
보안 = 4
악성코드 = 5
```

하지만 이 방식에는 큰 문제가 있다.

숫자 ID는 단어의 의미를 담지 못한다.

예를 들어 `보안 = 4`, `악성코드 = 5`라고 했을 때, 컴퓨터는 이 두 단어가 의미적으로 가깝다는 사실을 모른다.  
그냥 4와 5라는 번호만 볼 뿐이다.

---

## 2. One-hot Encoding의 한계

단어를 표현하는 가장 단순한 방식 중 하나는 **One-hot Encoding**이다.

단어가 5개라면 각 단어를 다음처럼 표현할 수 있다.

| 단어 | 벡터 |
|---|---|
| 사과 | [1, 0, 0, 0, 0] |
| 바나나 | [0, 1, 0, 0, 0] |
| 컴퓨터 | [0, 0, 1, 0, 0] |
| 보안 | [0, 0, 0, 1, 0] |
| 악성코드 | [0, 0, 0, 0, 1] |

이 방식은 단순하지만 치명적인 한계가 있다.

### 한계 1: 단어 간 의미적 유사도를 표현하지 못한다

One-hot 벡터에서는 모든 단어가 서로 동일하게 멀다.

`보안`과 `악성코드`도 멀고, `보안`과 `바나나`도 똑같이 멀다.

하지만 현실에서는 다음 관계가 더 자연스럽다.

```text
보안 ↔ 악성코드: 가까움
보안 ↔ 취약점: 가까움
보안 ↔ 바나나: 멂
```

One-hot Encoding은 이런 의미 관계를 표현하지 못한다.

### 한계 2: 단어 수가 많아질수록 벡터가 너무 커진다

단어가 100만 개라면 각 단어 벡터도 100만 차원이 된다.

대부분의 값은 0이고, 딱 하나만 1이다.  
이런 벡터를 **sparse vector**라고 한다.

LLM이나 딥러닝 모델에서는 이런 방식이 비효율적이다.

---

## 3. 임베딩이란 무엇인가?

임베딩은 단어를 **의미를 가진 숫자 벡터**로 바꾸는 방식이다.

예를 들어 `보안`이라는 단어를 다음처럼 표현할 수 있다.

```text
보안 = [0.21, -0.15, 0.83, 0.04, ...]
```

`악성코드`도 비슷한 벡터를 가질 수 있다.

```text
악성코드 = [0.25, -0.11, 0.79, 0.02, ...]
```

이 두 벡터가 비슷한 방향을 가진다면, 모델은 두 단어가 의미적으로 관련 있다고 볼 수 있다.

핵심은 이것이다.

```text
단어의 의미를 벡터 공간의 위치로 표현한다.
```

즉, 단어는 더 이상 단순한 ID가 아니다.  
각 단어는 고차원 공간 안의 한 점이 된다.

---

## 4. 분포 가설: 의미는 주변 단어에서 온다

Word2Vec의 핵심 아이디어는 **분포 가설(distributional hypothesis)** 에 기반한다.

간단히 말하면 다음과 같다.

```text
비슷한 문맥에서 등장하는 단어는 비슷한 의미를 가진다.
```

예를 들어 다음 문장을 보자.

```text
악성코드가 시스템을 감염시켰다.
랜섬웨어가 시스템을 감염시켰다.
트로이목마가 시스템을 감염시켰다.
```

`악성코드`, `랜섬웨어`, `트로이목마`는 비슷한 주변 단어와 함께 등장한다.

```text
시스템
감염
탐지
차단
분석
```

그래서 모델은 이 단어들이 의미적으로 가깝다고 학습할 수 있다.

이 관점이 매우 중요하다.

LLM도 결국 대규모 텍스트에서 단어와 문맥의 패턴을 학습한다.  
물론 Word2Vec보다 훨씬 복잡하고 강력하지만, 기본 철학은 이어진다.

---

## 5. Word2Vec이 하려는 일

Word2Vec의 목표는 단어를 의미 있는 벡터로 바꾸는 것이다.

다만 사람이 직접 의미를 정의하지 않는다.

예를 들어 사람이 직접 이렇게 정하지 않는다.

```text
보안은 악성코드와 가깝다.
사과는 바나나와 가깝다.
보안은 바나나와 멀다.
```

대신 많은 문장을 보고 자동으로 학습한다.

```text
주변 단어를 잘 예측할 수 있도록 단어 벡터를 조정한다.
```

즉, Word2Vec은 단어의 뜻을 사전처럼 외우는 게 아니라, **문맥 예측 문제를 풀면서 단어 벡터를 학습**한다.

---

## 6. Word2Vec의 두 가지 방식

Word2Vec에는 대표적으로 두 가지 학습 방식이 있다.

```text
1. CBOW
2. Skip-gram
```

---

## 7. CBOW: 주변 단어로 중심 단어 예측하기

CBOW는 **Continuous Bag of Words**의 약자다.

CBOW는 주변 단어들을 보고 가운데 단어를 예측한다.

예를 들어 다음 문장이 있다고 하자.

```text
악성코드가 시스템을 감염시켰다
```

여기서 중심 단어가 `시스템을`이라고 하면, 주변 단어는 다음과 같다.

```text
악성코드가, 감염시켰다
```

CBOW의 목표는 다음과 같다.

```text
[악성코드가, 감염시켰다] → 시스템을 예측
```

즉, 문맥을 보고 빠진 단어를 맞추는 방식이다.

### CBOW의 특징

CBOW는 여러 주변 단어를 평균적으로 보고 중심 단어를 예측한다.

그래서 비교적 빠르고 안정적이다.  
하지만 희귀 단어의 의미를 세밀하게 학습하는 데는 Skip-gram보다 약할 수 있다.

---

## 8. Skip-gram: 중심 단어로 주변 단어 예측하기

Skip-gram은 CBOW와 반대 방향이다.

중심 단어를 보고 주변 단어를 예측한다.

예를 들어 중심 단어가 `악성코드`라면, 주변에 나올 법한 단어를 예측한다.

```text
악성코드 → 탐지
악성코드 → 감염
악성코드 → 분석
악성코드 → 차단
```

Skip-gram의 목표는 다음과 같다.

```text
중심 단어 → 주변 단어 예측
```

### Skip-gram의 특징

Skip-gram은 희귀 단어에도 비교적 강하다.

예를 들어 `CVE-2024-xxxx`, `랜섬웨어`, `APT`, `C2 서버` 같은 보안 도메인 단어는 일반 문서에서는 자주 나오지 않을 수 있다.

이런 희귀 단어의 주변 문맥을 잘 학습하면 의미 있는 벡터를 얻을 수 있다.

---

## 9. Window Size란 무엇인가?

Word2Vec에서 중요한 개념 중 하나가 **window size**다.

Window size는 중심 단어 기준으로 주변 몇 개 단어까지 볼 것인지를 의미한다.

예를 들어 문장이 다음과 같다고 하자.

```text
사용자는 악성코드를 탐지하고 시스템을 격리했다
```

중심 단어가 `탐지하고`이고 window size가 2라면 주변 단어는 다음과 같다.

```text
악성코드를, 시스템을
```

window size가 3이라면 더 넓게 본다.

```text
사용자는, 악성코드를, 시스템을, 격리했다
```

Window size가 작으면 가까운 문맥을 학습한다.  
Window size가 크면 더 넓은 주제적 관계를 학습한다.

---

## 10. 벡터 공간에서 의미가 생긴다는 것

Word2Vec이 유명해진 이유 중 하나는 단어 간 관계가 벡터 연산으로 어느 정도 표현된다는 점이었다.

가장 유명한 예시는 다음과 같다.

```text
king - man + woman ≈ queen
```

한국어로 표현하면 이런 느낌이다.

```text
왕 - 남성 + 여성 ≈ 여왕
```

보안 도메인으로 비슷한 예를 상상해볼 수도 있다.

```text
랜섬웨어 - 암호화 + 정보탈취 ≈ 스틸러
```

물론 항상 정확하게 동작하는 것은 아니다.  
하지만 중요한 점은 단어의 의미 관계가 벡터 공간 안에서 어느 정도 구조를 가진다는 것이다.

---

## 11. Word2Vec과 현대 Embedding 모델의 차이

Word2Vec은 단어 단위 임베딩이다.

즉, 하나의 단어는 보통 하나의 고정된 벡터를 가진다.

예를 들어 `bank`라는 단어가 있다고 하자.

```text
bank = 하나의 벡터
```

하지만 `bank`는 문맥에 따라 의미가 다르다.

```text
river bank = 강둑
bank account = 은행 계좌
```

Word2Vec은 이런 문맥별 의미 차이를 잘 반영하지 못한다.

반면 BERT, GPT 같은 Transformer 기반 모델은 문맥에 따라 토큰 표현이 달라진다.

```text
같은 단어라도 문장 안에서 다른 벡터 표현을 가질 수 있다.
```

이걸 **contextual embedding**이라고 한다.

---

## 12. Static Embedding vs Contextual Embedding

| 구분 | Static Embedding | Contextual Embedding |
|---|---|---|
| 대표 모델 | Word2Vec, GloVe, FastText | BERT, GPT, RoBERTa |
| 단어 표현 | 단어마다 고정 벡터 | 문맥에 따라 달라짐 |
| 장점 | 빠르고 단순함 | 의미 구분이 강함 |
| 한계 | 다의어 처리 약함 | 계산 비용 큼 |

Word2Vec은 static embedding이다.

즉, `취약점`이라는 단어는 어떤 문장에 있어도 대체로 같은 벡터를 가진다.

하지만 Transformer 기반 모델에서는 문맥에 따라 달라질 수 있다.

```text
소프트웨어 취약점을 패치했다.
조직의 운영 취약점을 개선했다.
```

두 문장의 `취약점`은 비슷하지만 완전히 같은 의미는 아니다.  
Contextual embedding은 이런 차이를 더 잘 반영할 수 있다.

---

## 13. RAG에서 Embedding을 볼 때 연결되는 개념

네가 다루는 RAG, pgvector, 보안 문서 검색과도 이 개념은 직접 연결된다.

RAG에서는 문서 chunk를 벡터로 바꾼다.

```text
문서 chunk → embedding vector → vector DB 저장
```

사용자 질문도 벡터로 바꾼다.

```text
사용자 질문 → embedding vector
```

그 다음 질문 벡터와 가까운 문서 벡터를 찾는다.

```text
질문 벡터와 가까운 chunk 검색
```

이 방식의 뿌리에는 Word2Vec에서 시작된 관점이 있다.

```text
텍스트의 의미는 벡터 공간에서 거리와 방향으로 표현할 수 있다.
```

현대 embedding 모델은 Word2Vec보다 훨씬 복잡하지만, 기본 발상은 이어진다.

---

## 14. 보안 도메인 예시로 이해하기

보안 문서 임베딩을 만든다고 생각해보자.

다음 단어들은 의미적으로 가까운 위치에 있어야 한다.

```text
악성코드
랜섬웨어
스틸러
트로이목마
백도어
C2
IOC
```

다음 단어들도 가까운 영역에 있을 수 있다.

```text
취약점
CVE
패치
Exploit
PoC
CVSS
EPSS
```

반면 다음 단어들은 보안 문맥에서는 상대적으로 다른 영역에 있을 수 있다.

```text
마케팅
재무제표
농산물
여행
```

임베딩 모델이 좋다는 것은 단순히 벡터 차원이 크다는 뜻이 아니다.

좋은 임베딩 모델은 다음을 잘해야 한다.

```text
비슷한 의미의 텍스트는 가깝게 배치한다.
다른 의미의 텍스트는 멀게 배치한다.
질문과 관련 문서를 같은 공간에서 잘 연결한다.
```

## 15. Word2Vec이 LLM으로 이어지는 지점

Word2Vec은 LLM은 아니다.

하지만 LLM으로 이어지는 중요한 개념을 제공했다.

```text
단어를 숫자 벡터로 표현한다.
문맥을 기반으로 의미를 학습한다.
벡터 공간에서 의미 관계를 다룬다.
대량의 텍스트에서 언어 패턴을 학습한다.
```

LLM은 이 아이디어를 훨씬 더 큰 규모와 복잡한 구조로 확장한 것이다.

Word2Vec은 단어 중심이었다.  
Transformer는 문맥 전체를 본다.  
LLM은 대규모 문맥에서 다음 토큰을 예측하며 언어 능력을 학습한다.

```text
6. Skip-gram은 중심 단어로 주변 단어를 예측한다.

```
